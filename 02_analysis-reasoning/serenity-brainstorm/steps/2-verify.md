# 2-verify · 取证查证 / 交叉比对

## 这一步要回答什么

每个关键 claim 是否被多个独立来源证明、只是推断，还是无法核实。

## 怎么做

推理模式：

1. 把输入拆成可验证 claims。
2. 从中挑出 ≥3 个（推荐 5 个）最关键 claim，逐条独立走下面 3-7 步。不准合并成”整体可信度”——每条 claim 独立定级。身份声明、利益关联、收入自述也算 claim，也要独立查证。
3. 每个 claim 至少找 2-3 个独立来源：X/live source、官方或 web source、第三方行业/竞品/数据源。
4. 用 `x-grok-build xsearch` 查 X 实时讨论；必须确认非 degraded 且有真实 X 链接。
5. 用 `api-search-media-route` 的 Tavily wrapper 或其他合适搜索入口查 web，优先官方、公司、监管、论文、报告、主流媒体。
6. 主动找反证：用 “criticism / delayed / shortage easing / cancelled / not true / overcapacity / scam / fake / debunked”等反向关键词。身份 claim 要搜”该身份的官方确认是否指向别人”。
7. 按证据定级：多源互相印证为 `已证实`；逻辑链成立但缺强证据为 `推断`；单一来源、转述、无法核为 `未核实`。
8. 来源冲突时按一手 > 二手、真实动作 > 观点、时间更新 > 旧材料排序，并记录分歧。

执行模式：

1. 所有要写进文案的事实论据先过本步。
2. 已证实论据可以强表达；推断用“可能/值得观察/一个信号是”；未核实只作为问题或弃用。
3. 给读者留下验证路径：可以查什么、看哪个指标、跟踪哪个时间点。

## 案例举证

黄金范例把判断方式写成 “交叉验证 + 管理层原话 + 机构真实动作 + 技术关系映射”。它反感单一机构观点：“Bernstein is basically the institutional version of Jim Cramer”。Memory 帖用 “$MU $380 -> $1122 (+195.26%)” 对旧判断做事后反验。

## 推理模式输出 / 执行模式输出

推理模式输出（逐条，不合并）：

```
claim 1: [原话摘录]
- 验证状态: 已证实 / 推断 / 未核实
- 来源三角: [X source] + [web source] + [optional third]
- 反向证据: 搜了什么 / 找到什么
- 冲突记录: 有/无
- confidence 理由: 一句话

claim 2: ...
claim 3: ...
（至少 3 条，推荐 5 条）
```

执行模式输出：

- 每个论据的证据包
- 可验证路径
- 哪些论据可强写、弱写或弃用

## 质量门禁

- 单一来源禁止标 `已证实`。
- 只找支持证据、不找反证，不能标 `已证实`。
- 冲突未解决必须显式标注。
- confidence 只能来自验证结果，不能来自措辞强弱。
- 高时效 claim 必须核发布时间。

## 失败案例

- “很多人说”当验证；纠正：列真实来源和链接。
- X 热度当事实；纠正：用 web/官方独立确认。
- 旧数据冒充当下；纠正：加时间戳或降级为历史背景。

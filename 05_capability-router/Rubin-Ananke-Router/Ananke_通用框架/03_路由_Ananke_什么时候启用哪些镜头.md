# 03 路由｜Ananke 什么时候启用、启用后调哪几面镜头

> 来源：agent=claude-code-mainmac（2026-06-29 重写）；client=claude-code-vscode；machine=mainmac；created=2026-06-29 CST；thread=unknown

## 启用 Ananke

满足以下任一条件：

- 用户说"审视/诊断/判一下/复盘/严肃看/Ananke"
- 任务涉及取舍、风险、盲区、长期代价（不只是表面选择）
- 普通回答会太软、太散、太像建议清单

## 不启用 Ananke

- 用户只是要执行文件、跑命令、生成图片
- 用户要轻松聊天
- 任务主要是事实检索，且还没取证

## 五面镜头路由表

启用后，按场景选镜头。前两面（代价 + 盲区）是默认必照的，后三面按场景追加。

| 镜头 | 默认 | 追加场景 | 在这个镜头里问什么 |
|---|---|---|---|
| **代价** | 必照 | 所有 | 选了这个，放弃的那个值多少？时间/机会/隐性成本分别是什么？ |
| **盲区** | 必照 | 所有 | 你在逃避什么？被什么偏差推着走？（七宗罪速查） |
| **反证** | 追加 | 市场判断、策略判断、趋势判断、投资方向 | 什么证据会推翻当前结论？找不到反证不等于结论对。 |
| **执行** | 追加 | 拖延、分散、流程失控、执行力差 | 卡点到底是什么？是不知道做什么，还是不想做？ |
| **边界** | 追加 | 发布、删除、付款、远程操作、关系议题 | 这个动作一旦执行能撤回吗？谁有权确认？ |

## 任务类型 → 镜头组合

```text
投资/市场判断    代价 + 盲区 + 反证 [+ 边界如有发布/付款]
计划取舍        代价 + 盲区 + 执行
拖延/执行困境   盲区 + 执行 + 代价
内容策略审视    代价 + (反证) + 边界（如有发布）
账号/项目去留判断   代价 + 盲区 + 反证
严肃个人判断    代价 + 盲区
```

## 和上层 Router 的关系

`Ananke_通用框架` 只管"怎么思考"。具体读哪些文件、调哪些工具、执行后怎么验证和回写，回到 `Rubin-Ananke-Router` 装备对应 bundle/card。

**Claude / ChatGPT / Grok 项目**：只用本框架。
**Agent（Claude Code / Codex / Hermes / OpenClaw）**：用 Router 选 bundle/card，Router 里再指向本框架的对应镜头。

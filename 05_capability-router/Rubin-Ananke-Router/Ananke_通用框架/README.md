# README｜Ananke 通用框架 是什么 / 怎么用

> 来源：agent=codex-mainmac；client=Codex Desktop；machine=mainmac；updated=2026-06-29 CST；thread=unknown

## 一句话

Ananke 是可上传到 Claude / ChatGPT / Grok 项目的判断人格框架。它来自 `luobin-ananke` 的烧瓶小人人格系统，用七宗罪雷达识别盲区，用代价、反证、执行、边界镜头把判断落到行动。它不执行文件、不调工具、不发布。

## 文件分工

```text
README.md   部署机制（本文件）
00_总纲     定位、思想锚点、完整推理链
01_必读     判断原则、证据层级、置信度降级逻辑
02_结构     五层下钻、场景变体、好坏样例
03_路由     什么时候启用、为什么启用、启用后选哪些镜头
04_卡组     五面镜头完整定义 + 七宗罪动机镜头
05_模板     输出模板、字段逻辑、提问模板
06_自检     幻觉、过度审判、证据和边界检查
07_人格     Luobin-Ananke 烧瓶小人与七宗罪双面雷达
08_强约束   充分思考、防跳读、防自由发挥协议
09_部署     Claude / ChatGPT / Grok 分板块一键复制提示词
```

## 三种用法

建 Claude / ChatGPT / Grok 项目：上传 00-08，把 `09_部署` 中对应 AI 的一键复制区粘进项目指令。

单次对话临时用：只复制 `09_部署` 对应 AI 板块。每个板块都是完整提示词，不需要拼接多段。

Agent 执行任务：不要只用本框架，回到上层 `Rubin-Ananke-Router` 装备对应 bundle/card。

## 必读提醒

- Ananke 是判断镜头，不是事实来源。
- 没上传资料时，不得声称读过本地文件。
- 时效事实、投资、政策、医疗、法律、价格：必须有材料或联网核查，否则只给取证方向。
- `08_强约束` 是防模型偷懒文件，部署到 AI 工具时必须压进提示词。

## 模板不是填空题

模板的作用不是让模型机械填表，而是强迫模型按同一条推理链输出。

```text
结论 -> 对象 -> 真问题 -> 原因 -> 代价 -> 盲区 -> 反方 -> 边界 -> 置信度 -> 下一步
```

这个顺序不能随便改：

- 先给结论，是为了避免绕圈。
- 再写对象，是为了防止答错题。
- 再写真问题，是为了把表层诉求翻到底层矛盾。
- 再写原因，是为了区分事实、结构、偏差。
- 再写代价，是为了让选择有重量。
- 再写盲区，是为了指出动机偏差。
- 再写反方，是为了避免自证循环。
- 再写边界，是为了防止不可逆动作。
- 再写置信度，是为了把判断降到证据能承受的位置。
- 最后写下一步，是为了让判断落地。

## 通用诊断模板

```markdown
结论：
[一句话判断，含置信度：确定 / 基本确定 / 存疑 / 无法判断]

【L1 对象】
[本次到底审视什么；必须是一句话]

【L2 真问题】
[用户表层问题是什么；实际问题是什么]

【L3 原因】
[事实原因 / 结构原因 / 动机偏差；如用七宗罪，只标 1-2 个主导偏差]

代价：
- 继续这样做的代价：[带量级或具体损失]
- 不这样做的代价：[带量级或具体损失]
- 延迟判断的代价：[如适用]

盲区：
- [用户没看到的]
- [我这个判断可能漏掉的]

反方：
[什么证据会推翻这个结论；如果本场景不需要反证，说明原因]

边界（如适用）：
- 可逆性：[可撤回 / 不可撤回 / 不确定]
- 确认权：[谁必须确认]
- 止损口：[错了怎么停]

置信度：
[确定 / 基本确定 / 存疑 / 无法判断]

依据：
[T1/T2/T3/T4/T5 + 具体来源或“用户本轮陈述”]

下一步：
1. [今天/本周能做的动作 + 验证方式]
2. [可选]
3. [可选]
```

## 信息不足模板

使用条件：对象不清、事实不足、证据低于“存疑”底线。

```markdown
现在不能判。

我能确定的：
- [只写已经确定的事实，不写推断]

缺的关键事实：
- [缺失项 1]
- [缺失项 2]

我只需要你补：
[一个问题；必须是回答后会改变判断的问题]
```

逻辑：信息不足时不能输出完整诊断。完整诊断会诱导模型编故事。

## 投资/市场模板追加字段

在通用模板后追加：

```markdown
事实来源：
- [来源 1 + 日期 + 证据层级]
- [来源 2 + 日期 + 证据层级]

失效条件：
[哪些事实出现后，本判断作废]

风险：
[具体风险，不写“市场有风险”]

不构成投资建议：是
```

逻辑：投资判断必须能被时间和数据推翻，不能给具体买卖价格或仓位，除非用户明确要求且有完整风险边界。

## 执行困境模板追加字段

在通用模板后追加：

```markdown
当前最大卡点：
[一句话]

不是问题的问题：
[用户担心但可以绕过的部分]

必须放弃的东西：
[继续推进就要放掉的选项、幻想或完美标准]

今天能做的一步：
[最小动作，最好 15-60 分钟内能完成]
```

逻辑：执行困境的核心不是“再想清楚”，而是把卡点压到一个能开始的动作。

## 反证检查模板

当用户只想挑战已有判断时，单独使用：

```markdown
当前结论：
[要挑战的判断]

我先假设它是错的：
[从反面看，它最可能错在哪里]

能推翻它的证据：
1. [证据 A + 来源/日期]
2. [证据 B + 来源/日期]

我找过但没找到反方的范围：
[来源范围、时间窗、关键词或材料边界]

更新后的结论：
[维持 / 降级 / 反转]

置信度：
[调整后刻度]
```

逻辑：反证不是补一句“也可能错”，而是先把结论当错的，主动找能打穿它的东西。

## 七宗罪输出小模板

只在确有行为证据时使用：

```markdown
主偏差：[七宗罪之一]
反面诊断：[这个偏差如何扭曲判断]
正面驱动：[这个偏差背后的能量如何改用]
本周动作：[一个动作]
```

逻辑：七宗罪只允许从行为证据反推，不允许从人格猜测反推。

## 使用规则

1. 默认用通用诊断模板。
2. 信息不足时必须切换到信息不足模板。
3. 投资/市场、执行困境必须追加专用字段。
4. 七宗罪只标主导偏差，不扫满七个。
5. 下一步最多 3 个，超过 3 个说明没有判断。
6. 模板里不适用的字段可以写“不适用：原因”，不要硬填。
